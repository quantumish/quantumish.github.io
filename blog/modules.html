What's a module?

The more formal definition of a module is as follows.

Let \(R\) be a commutative ring with an identity. A module over \(R\) (also known as a \(R\text{-module}\)) is a nonempty set \(M\) under two different operations. The first operation, addition, assigns to each pair \((u,v) \in M \times M\) an element \(u+v \in M\). The second operation, juxtaposition, assigns to each pair \((r,u) \in R \times M\)This technically makes this a left module, whereas a right module would have some juxtaposition operation bringing \(M \times R \to M\) an element \(rv \in M\).

Furthermore, the following conditions must hold:

  1. \(M\) is an abelian group under addition.
  2. For all \(r,s \in R\) and \(u,v \in M\):
    • \(r(u+v)=ru+rv\)
    • \((r+s)u = ru+su\)
    • \((rs)u = r(su)\)
    • \(1u = u\)

What does that mean?

That was a lot of word soup, so let's break down some of the terminology first.

An abelian group is a group that is commutative.
A ring is an abelian group under addition with a second operation, called multiplication, that is associative, distributive, and has an identity.
Essentially, a ring is a field without a requirement of commutativity or a multiplicative inverse. However, we're operating on a commutative ring and thus the only real change is the removal of the multiplicative inverse.

With these definitions and thinking of juxtaposition as fancier scalar multiplication, we can consider modules as vector spaces where the scalars need not have multiplicative inverses...but what do we even get from that? Well, there are some familiar sets that we can't have vector spaces over but can have modules over:
  • Our friends the integers, \(\mathbb{Z}\).
  • The set of \(n\)-degree polynomials \(P_n\).
Wouldn't it be nice to be able to apply all of our linear algebra logic to these sets? I mean, we don't really divide things all too much in linear algebra, so surely it'd be fine to remove multiplicative inverses, right?

Linear Dependence

Let's take a simple example: the module \(\mathbb{Z}\) over the ring \(\mathbb{Z}\) (since we now know we can do this). Thinking back to our work with linear dependence, we know that some vector in a linearly dependent list is in the span of the previous vectors (via the Linear Dependence Lemma).

Thus, if we consider two vectors not in the span of one another (in the context of \(\mathbb{Z}\)), like the numbers 3 and 2, we should not be able to pick \(a_1, a_2\) such that their linear combination \(a_1v_2 + a_2v_2 = 0\). Yet, we can easily find a counterexample. \begin{align*} \text{Let }a_1, a_2 = 2, -3\text{ and }v_1,v_2 = 3,2.\\ 2(3)-3(2)=0 \end{align*} Uh oh. Say goodbye to the Linear Dependence Lemma — if you think back to our linear algebra knowledge, you might even remember proving it using scalar division (which is obviously not possible anymore)!

Aside: proving the Linear Dependence Lemma.

If a list \(v_1, \dots, v_m\) is linearly dependent, there must exist scalars \(a_1, \dots, a_m\) such that \(a_1v_1 + \cdots + a_mv_m = 0\).
Let \(j\) be the largest element of \(\{1,\dots,m\}\) such that \(a_j\) is nonzero. \begin{align*} a_1v_1 + \cdots + a_jv_j = 0 \\ a_1v_1 + \cdots + a_{j-1}v_{j-1} = -a_jv_j \\ -\frac{a_1}{a_j}v_1 - \cdots - \frac{a_{j-1}}{a_j}v_{j-1} = v_j \\ \end{align*} Thus, \(v_j\) is expressible as a linear combination of \(v_1, \dots, v_{j-1}\) and is in the span of that list. Such logic does not apply to modules, as our scalars come from rings, which do not necessarily have a multiplicative inverse. As a result, we can't do these algebraic manipulations and our Linear Dependence Lemma breaks down.


We use the Linear Dependence Lemma for proving quite a lot, and now none of those things apply to modules! Some brief examples:
  • The lengths of linearly independent lists are no longer necessarily less than or equal to the lengths of spanning lists.
  • Spanning lists no longer have to contain a basis.
  • Linearly independent lists don't necessarily extend to a basis.

Torsion Elements

Modules are able to have no linearly dependent elements.

After some thought, this may prompt you to wonder how this is even possible, as a list containing a single nonzero will always be linearly independent... right? Nope. No longer applicable.

New example: take the module \(\mathbb{Z}/6\)This denotes the set of all integers modulo six, which is also sometimes represented as GF(6). over the ring \(\mathbb{Z}\). Any element \(\mathbb{Z}\) times 6 (or any multiple of 6) will equal zero because of the properties of modular arithmetic. This means that there exist \(r \in \mathbb{Z}\) such that \(r \neq 0\) yet \(rz = 0\) for any \(z \in \mathbb{Z}\). In plain english, there are scalars other than zero that when multiplied with any vector always equal zero. As a result, we can always have nonzero linear combinations of a single vector \(v\) (aka scalar multiples of \(v\)) that are equal to zero, making them linearly dependent.

These terrible scalars are called torsion elements, and the set of all of them is called the annihilator.No relation to the other annihilator in linear algebra, which has to do with linear functionals. This of course dismantles more of our expectations about modules, as they no longer need to have bases and thus anything that depends on bases no longer apply as well.

Submodules and Finitely-Generated Modules

A submodule is a subset of a module that is also a module with respect to the same ring. More formally, a submodule of an \(R\)-module \(M\) is a nonempty subset \(S\) of a module \(M\) that is also an \(R\)-module under the operations obtained by restrcting the operations of \(M\) to \(S\).
We switch up our vocabulary a little when discussing submodules:
  • Instead of saying a list spans a module, we say the module generated by a list is the set of all linear combinations of elements of that list.
  • Instead of referring to modules as finite-dimensional if they are generated by a finite list, we say they are finitely-generated.
